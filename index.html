  • IM.js checks all your images using canvas pixel by pixel in record time.

    History:

    One month ago I needed to test the rendering url results in different browsers, and make it automatic and use it with Jenkins (C.I. server).
    I found some desktop tools to compare images but that are not for use with Jenkins.
    I found some Javascript classes that only check the MD5 and Base65 encoding for the image.

    This was not useful, because I didn't want to check different image files.
    I want to check different images seeking for different pixel info, because different compression returns different image file info.

    Description:

    Compare images pixel by pixel is a slow and heavy work to our computer!
    IM.js has come to solve this problem!

    Designed to:

    • Compare images pixel by pixel in a fast and not blocking way.
    • Be used in environments to automatize testing of web url snapshots.
    • Be framework agnostic.

    Install:

    <script type="text/javascript" src="/path/to/IM.js"></script>

  • IM.js API

    IM.setDebug(bDebug)

    Sets the debug mode.

    When debug mode is on, the class sends info about how many time tooks the comparison. And if for some reason it returns that one image is different, the canvas is hightlighted with a red border.

    Params:
    • bDebug : {Boolean} true or false (Default is false).

    IM.setAsynchronous(bAsync)

    Sets the asynchronous mode.

    When asynchronous mode is on, the loop will be launched in a non blocking system. Asynchronous mode must be used only if there are some problems with browser performance.

    Params:
    • bAsync : {Boolean} true or false (Default is false).

    IM.image

    Is a reference to the config object for images.

    new IM.image(sPath, nWidth, nHeight)

    Params:
    • sPath : {String} Url/src to the image to be loaded.
    • nWidth : {Number} Width of the image.
    • nHeight : {Number} Height of the image.

    IM.compare(DOM_Element, aImages, fpSucessCallback, fpFailCallback)

    This compare mode creates the canvas inside the DOM_Element containing the aImages. If all the images are the same the fpSuccessCallback will be executed, if any of the image is different the fpFailCallback will be executed.

    Params:
    • DOM_Element : {DOM} Element where add all the needed canvas (one canvas per image).
    • aImages : {Array} Array of config image objects (see IM.image).
    • fpSucessCallback : {Function} Callback that will be triggered if all the images are equal.
    • fpFailCallback : {Function} Callback that will be triggered if any of the images is different.

    IM.compare(aCanvas, fpSucessCallback, fpFailCallback)

    This compare mode gets an array of canvas containing the aImages. If all the images are the same the fpSuccessCallback will be executed, if any of the image is different the fpFailCallback will be executed.

    Params:
    • aCanvas : {Array} Array of canvas with images to be compared.
    • fpSucessCallback : {Function} Callback that will be triggered if all the images are equal.
    • fpFailCallback : {Function} Callback that will be triggered if any of the images is different.

    IM.showDiffInCanvas(oDOM)

    This method creates a canvas where the two different images are joined to see the differences at one sight.

    Params:
    • oDOM : {DOM} Element where append the canvas.

    IM.setTolerance(nPercentage)

    This method sets a tolerance percentage to mark an image as ok or ko.

    Params:
    • nPercentage : {Number} Number of percentage.

  • Code examples

    Sets debug mode on:
    	                    			
  IM.setDebug(true);
    Sets debug mode off:
    	                    			
  IM.setDebug(false);
    Set asynchronous mode on:
    	                    			
  IM.setAsynchronous(true);
    Set asynchronous mode off:
    	                    			
  IM.setAsynchronous(false);
    Set asynchronous mode on:
    	                    			
  IM.setAsynchronous(true);
    Create a new config image object:
    	                    			
  new IM.image('path/to/images/image.png', 100, 100);
    Compare images creating canvas and images using config image objects:
    	                    			
  IM.compare(document.body,
	[
		new IM.image('path/to/images/image.png', 100, 100),
	       	new IM.image('path/to/images/image2.png', 100, 100),
	       	new IM.image('path/to/images/image.png', 100, 100),
	       	new IM.image('path/to/images/image2.png', 100, 100)
	],
	function success(aCanvas, nElapsedTime, nPercentageMatch) {
	  // Code on success. All images have the same pixel info.
	},
	function fail(oCanvas, nElapsedTime, nPercentageMatch) {
	 // Code on failing. Any image is different from others
	});
    Compare images using existent canvas that contains images:
    	                    			
  IM.compare(
	[
		document.getElementsById("canvas_1"),
	       	document.getElementsById("canvas_2"),
	       	document.getElementsById("canvas_3"),
	       	document.getElementsById("canvas_4")
	],
	function success(aCanvas, nElapsedTime, nPercentageMatch) {
	  // Code on success. All images have the same pixel info.
	},
	function fail(oCanvas, nElapsedTime, nPercentageMatch) {
	 // Code on failing. Any image is different from others
	});
    Create a canvas object with the result of differences in images:
    	                    			
  IM.showDiffInCanvas(document.body);
    Set a tolerance percentage value to mark an image as ok or ko:
    	                    			
  IM.setTolerance(90);
